Exercise 1: Personal Class Website with Homepage

Your personal class website will be used to store and present all exercises, journal entries and projects. It is your personal space so it should evolve over the course of the semester as your knowledge grows. It can be designed any way you'd like, the only requirements are:

  • A homepage which includes links to exercises, journal entries and projects

We'll be using the GitHub developer platform to host the websites. The website will be stored in a "repository" which you can think of as a folder on your computer.

  1. Create an account on GitHub (make sure to verify the account) using your Fordham email.
  2. After you're logged in, go to your main repositories page. Click the NEW button.
  3. Name your repository whatever you would like (something like “class-site” if you can’t think of anything).
  4. Select PUBLIC (Anyone on the internet can see this repository).
  5. Check Add a README file.
  6. Leave other options as-is.
  7. Click CREATE REPOSITORY.
  8. You are now viewing the contents of the repository which is currently empty except for the README file. A README file is simply a description of the contents of the respository. Click the edit icon if you'd like to change the text of the README file.

In order for your repository to be treated and rendered like a website, you need initialize GitHub Pages on this repository. Github pages is a feature offered by github that makes it easy and free to host handmade websites.

  1. Click SETTINGS in the top navigation.
  2. Click PAGES in the left navigation.
  3. Under BRANCH, select MAIN. Click the SAVE button.
  4. Wait a few minutes and refresh the page. You will see your repo link: your-github-username.github.io/class-site

As you complete exercises and projects, you will add these files to the repo. In this exercise, we will create a homepage (index.html) but going forward, the process is:

  1. Click CODE in the top navigation.
  2. Click ADD FILE > UPLOAD FILES.
  3. Drag the new files or folders to the area indicated.
  4. COMMIT CHANGES - this is where you add some short description of what you've changed to keep track of your activity (something like "first upload", "updated homepage", "add new project", etc.)
  5. Wait a few minutes and then check your repo link.
  6. Add your URL to the spreadsheet.

Note: GitHub has many features, including a desktop app that automatically updates your repo. We will not be using the app but feel free to try out these features on your own computer.

Exercise 2

Use this text to recreate Ten Principles for Good Design by Dieter Rams. Use what you know about HTML and CSS (also, refer to the resources) and follow these instructions:

  • One page only, a stylesheet and a folder for images
  • Include a minimum of five images which demonstrate the principles in your opinion (be creative, do not use Dieter Ram images, you can also create your own images). These can be placed anywhere that makes sense within your design.
  • Use as many HTML tags and CSS properties as you'd like.
  • In terms of design, think about what the choice of fonts, colors and placement of images says about your design.

Recommended Process:

  1. Create a local folder "exercise2". Folder should contain index.html, a stylesheet and a sub-folder for images.
  2. When you are ready, upload this entire folder to your GitHub class website. The link should look something like: your-github-username.github.io/class-site/exercise2/. You can continue to edit and re-upload.
  3. Add your URL to the spreadsheet.

Note: Exercises are not graded but you should hand them in on time.
